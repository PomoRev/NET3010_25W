Data Structures for the crossword puzzle.

puzzle

- solvedPuzzle is an array of the solved puzzle generated by makePuzzle() 
  and used to initialize the displayPuzzle as well as to check the displayPuzzle 
  for completion (win condition: all the guessedLetter values are the same as the 
  correctLetter values - put in function isWon() ). A 2 dimensional array with 
  elements containg both a cellNumber (default NONUMBER) and a 
  correctLetter (default NOLETTER)

- guessedPuzzle a 2 dimensional array with elements containg both 
  a cellNumber (default NONUMBER) and a guessedLetter (default NOLETTER)

- lists of clues (cluesAcross and cluesDown - both are arrays of strings (HTML)

- timer which holds the time spent on current puzzle
    - start time current session
    - time elapsed already (default 0)

- timerInervalID is a function for running the timer without hogging the thread (defualt NULL)

- currentWordNumber (start at 1)

- currentDirection is the direction we are currently working and can be toggled (default ACROSS)

- currentClickedElement is a reference to an actual cell on the puzzle that has been clicked (default NOTCLICKED)


wordList (word_data.json => converted to a JS Object for ease of use)

constant values

NOLETTER - token for an empty square on our puzzle ('?')
NONUMBER - token for a square that has no number to point to a clue (0)
NOBESTTIME - token for an empty best time record
NUMWORDS - number of words to try to add to puzzle
MAXWIDTH - width of the puzzle in character spaces
MAXHEIGHT - height of the puzzle in characters spaces
NOTCLICKED - constant for currently selected puzzle space (-1)
ACROSS - to set the directionality for puzzle creation and interface actions (true)
DOWN - to set the directionality for puzzle creation and interface actions (false)
NOCELLSELECTED - constant for not having any starting point for typing letters (-1)

query selectors

'#puzzle table tr td' - grab every cell for the current puzzle board (used for consistency)
